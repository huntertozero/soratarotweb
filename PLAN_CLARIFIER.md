# 추가 카드 기능 — Clarifier Card

**작성일**: 2026-06-01  
**최종 수정**: 2026-06-01  
**상태**: 구현 완료 (Phase 45) — 통합 해석 방식으로 전면 재설계

---

## 개념

카드 선택 완료 후, 오라클 구체가 실행되기 전에 특정 조건이 감지되면 사용자에게 **추가 카드 1~2장**을 선택할 기회를 제공한다.  
추가 카드는 별도 해석을 생성하는 것이 아니라, 원래 카드와 함께 **단일 통합 해석**에 포함된다.  
타로 관행에서는 "클라리파이어 카드(Clarifier Card)" 또는 "명확화 카드"라고 불리며, 이미 뽑힌 카드를 보강하는 역할을 한다.

---

## 사용자 플로우

```
SHUFFLE에서 카드 선택 완료
  → CARD_REVEAL: 원래 카드 플립
  → 조건 체크 (A/B/D)
     ├ 조건 있음: #clarifier-before-reading 표시
     │   → 사용자 추가 카드 N장 선택
     │   → "추가 카드 선택 완료" 버튼 클릭
     └ 조건 없음: 즉시 오라클 시작
  → 오라클 구체 + /api/reading 호출 (clarifierCards 포함)
  → READING: 원래 카드 + 추가 카드 통합 해석 표시
             추가 카드는 카드 목록 맨 오른쪽, '+' 뱃지
```

---

## 활성화 조건

조건 감지는 **모두 클라이언트**에서 수행한다. 서버 응답 불필요.

| 조건 | 감지 위치 | 발동 타이밍 | 적용 스프레드 | 카드 수 |
|------|----------|-----------|-------------|---------|
| A. 선택/비교 질문 | 클라이언트 | 카드 플립 완료 후 | 원 카드·3카드 | 2장 |
| B. 원 카드 역방향 | 클라이언트 | 카드 플립 완료 후 | 원 카드 | 1장 |
| D. 역방향 과반수 | 클라이언트 | 카드 플립 완료 후 | 3카드 | 1장 |

**우선순위**: A > B > D  
**켈틱 크로스(10장)**: 모든 조건 비허용

### 조건 A: 선택/비교 질문
- **정규식**: `/둘\s*중|어느\s*쪽|[가-힣]+와\s*[가-힣]+\s*중|아니면|vs\.?|선택해야|갈아타야/i`
- **이유 문구**: "선택지가 있는 질문에는 각 방향을 보여주는 카드가 도움이 돼요"
- **카드 수**: 2장 (각 선택지에 1장씩)

### 조건 B: 원 카드 역방향
- **조건**: `spread === 'one' && selectedCards[0].isReversed === true`
- **이유 문구**: "막혀있는 에너지를 뚫어줄 돌파구 카드가 있을 것 같아요"
- **카드 수**: 1장

### 조건 D: 역방향 과반수
- **조건**: `reversedCount / totalCards > 0.5` (3카드: 2장 이상 역방향)
- **이유 문구**: "막혀있는 에너지가 많아요. 흐름을 도와줄 카드가 있을 것 같아요"
- **카드 수**: 1장

> **삭제된 조건**
> - ~~조건 C (AI 불확실성 신호)~~: 해석 전에 감지 불가능하여 제거
> - ~~조건 E (아르카나 대립 패턴)~~: 구현 대비 체감 효과 불명확, 미구현

## API 설계

### POST /api/reading — clarifierCards 파라미터 추가

```json
요청:
{
  "spread": "three",
  "question": "A와 B 중 어느 것이 나을까요?",
  "cards": [...],
  "clarifierCards": [          // 선택사항 (없으면 빈 배열)
    { "id": 10, "isReversed": false },
    { "id": 11, "isReversed": true }
  ]
}

응답:
{
  "reading": "...(통합 해석 텍스트, '✦ 추가 카드' 섹션 포함)...",
  "cards": [...],              // 원래 카드 정보
  "clarifierCards": [...]      // 추가 카드 정보 (없으면 [])
}
```

- 켈틱 크로스: `clarifierCards` 자동 무시 (빈 배열로 강제)
- 중복 카드 ID 방어: 원래 카드 ID 집합에 추가 카드 ID 포함 여부 검증
- max_tokens: 기존 + 600 (최대 4096)

### 삭제된 엔드포인트
- ~~`POST /api/reading/clarifier`~~: Phase 45에서 삭제, `/api/reading`에 통합

## 프롬프트 설계

`prompts/one.md`, `prompts/three.md` 말미에 추가된 지시:

```
### 추가 카드가 포함된 경우 (선택)
프롬프트에 "✦ 추가 카드" 섹션이 있을 때만 적용하라. 없으면 이 지시는 무시하라.
- 위의 해석 구조를 모두 완성한 뒤, 마지막에 별도 섹션을 추가하라.
- 제목: **✦ 추가 카드: 영문명 (한글명)**
- 추가 카드의 이미지를 1문장으로 묘사하고, 앞선 해석에 더하는 메시지를 2~3문장으로 작성하라.
- 추가 카드까지 포함한 전체 방향을 따뜻한 한 문장으로 마무리하라.
```

Claude 프롬프트에 전달되는 추가 카드 섹션 포맷 (`formatClarifierCardsForPrompt()`):

```
**✦ 추가 카드 (클라리파이어)**
**카드: {name} ({nameKo}) {direction}**
- 핵심 키워드: ...
- 이미지 묘사: ...
```

## UI 설계

### CARD_REVEAL 화면 — `#clarifier-before-reading`

- `#clarifier-before-reading` 섹션 (`display:none` → 조건 충족 시 표시)
  - 제목: "✦ 추가 카드를 뽑아볼까요?" (sky-400 `#38bdf8` 색상)
  - 이유 문구 (문장 단위 `<br>` 줄바꿈)
  - 카드 선택 카운터: "N / N장"
  - `#clarifier-pre-grid`: 원래 선택 카드 제외한 나머지 `.card-back-item`
  - "추가 카드 선택 완료" 버튼 (선택 전 `disabled`)
  - 건너뛰기 없음 — 조건 발동 시 반드시 선택 후 진행

### READING 화면 — 카드 목록

- 원래 카드 + 추가 카드 → `allCards` 배열로 통합 렌더링
- 추가 카드는 목록 맨 오른쪽 배치 (`data-is-clarifier="1"`)
- `.clarifier-badge`: 우측 상단 `+` 뱃지, sky-400(`#38bdf8`) 색상, `clip-path` 화살표 (`.card-number-badge`의 좌우 미러)
- 추가 카드에는 위치 레이블 미표시

## 비용/성능

- 추가 API 호출 없음 — `/api/reading` 단일 호출
- 추가 카드 있을 때 max_tokens `+600` (토큰 증가분 ~300~500 토큰 예상)
- 24시간 제한: 기존 리딩 1회 제한에 포함 (별도 카운트 없음)
