# 추가 카드 기능 기획 — Clarifier Card

**작성일**: 2026-06-01  
**상태**: 구현 완료 (Phase 44) — 조건 A·B·C·D 모두 구현

---

## 개념

현재 리딩이 완료된 후, 특정 조건이 감지되면 사용자에게 **추가 카드 1~2장**을 선택할 기회를 제공하고 보충 해석을 생성한다.  
타로 관행에서는 "클라리파이어 카드(Clarifier Card)" 또는 "명확화 카드"라고 불리며, 이미 뽑힌 카드를 보강하는 역할을 한다.

---

## 활성화 조건

### 1단계 — 프론트엔드 단순 감지 (구현 우선)

#### 조건 A: 선택/비교 질문
- **감지 위치**: 클라이언트 (`app.js`), 질문 입력 완료 시
- **패턴 예시**: "A와 B 중", "~할까요 아니면", "둘 중 어느", "선택해야", "갈아타야"
- **정규식 초안**: `/둘\s*중|어느\s*쪽|[가-힣]+와\s*[가-힣]+\s*중|아니면|vs\.?/i`
- **플래그**: `questionType = 'comparison'`
- **설명**: 선택지가 명시된 질문은 각 선택지에 카드 1장씩 보강이 자연스럽다.

#### 조건 B: 원 카드 + 역방향
- **감지 위치**: 클라이언트, 카드 선택 확정 후
- **조건**: `spread === 'one' && selectedCards[0].isReversed === true`
- **설명**: 역방향 원 카드는 막힌 에너지를 의미하므로 "돌파구" 카드 1장 제안이 자연스럽다.

---

### 2단계 — AI 판단 기반 (Claude 응답 구조화)

#### 조건 C: AI 불확실성 신호
- **감지 위치**: 서버, Claude 응답 파싱
- **구현 방식**: 프롬프트에 아래 지시 추가
  ```
  해석이 끝난 후, 카드 에너지가 서로 충돌하거나 결말이 불명확하다고 판단되면
  응답 마지막에 다음 JSON을 한 줄로 추가하라:
  <!--CLARIFIER:{"needed":true,"reason":"충돌 에너지 - Tower와 Star"}-->
  ```
- **서버 파싱**: 응답 텍스트에서 `<!--CLARIFIER:{...}-->` 추출 후 제거, API 응답에 `clarifier` 필드로 분리
- **적용 스프레드**: 원 카드, 3카드만 (켈틱 크로스는 이미 10장으로 충분)
- **설명**: 가장 정확하지만 프롬프트 설계가 중요하다. 남용 시 무의미한 추가 호출 발생.

#### 조건 D: 역방향 과반수
- **감지 위치**: 서버, 카드 데이터 계산
- **조건**: `reversedCount / totalCards > 0.5`
  - 3카드: 2장 이상 역방향
  - 켈틱: 6장 이상 역방향 (켈틱은 조건 C 제외지만 이 조건은 허용)
- **설명**: 전체 에너지가 막혔을 때 "흐름 전환" 카드 1장 제안.

---

### 3단계 — 규칙 기반 패턴 (후순위)

#### 조건 E: 주요 아르카나 대립 조합
- **예시**: Tower(16) + Star(17), Death(13) + World(21), Hanged Man(12) + Judgment(20)
- **방식**: 사전 정의된 대립 쌍 목록으로 카드 ID 조합 매칭
- **우선순위**: 낮음 (구현 대비 체감 효과 불명확)

---

## 조건 우선순위 요약

| 조건 | 감지 위치 | 구현 난이도 | 적용 스프레드 | 단계 |
|------|----------|-----------|-------------|------|
| A. 선택/비교 키워드 | 클라이언트 | 낮음 | 전체 | 1단계 |
| B. 원 카드 역방향 | 클라이언트 | 낮음 | 원 카드 | 1단계 |
| C. AI 불확실성 신호 | 서버 (Claude) | 중간 | 원 카드, 3카드 | 2단계 |
| D. 역방향 과반수 | 서버 | 낮음 | 3카드, 켈틱 | 2단계 |
| E. 아르카나 대립 패턴 | 서버 | 높음 | 3카드 | 3단계 |

## API 설계 방향

### 기존 응답 확장
```json
POST /api/reading 응답:
{
  "reading": "...",
  "usage": { ... },
  "clarifier": {
    "needed": false,
    "reason": null,
    "trigger": null
  }
}
```

`trigger` 값: `"comparison"` | `"one_reversed"` | `"ai_signal"` | `"reversed_majority"` | `null`

### 추가 카드 리딩 엔드포인트 (신규)
```
POST /api/reading/clarifier
{
  "originalCards": [...],
  "clarifierCards": [...],   // 새로 뽑은 카드
  "question": "...",
  "spread": "one",
  "trigger": "comparison"
}
```

- 기존 해석 텍스트 없이 **보충 해석만** 생성 (max_tokens: 800, timeout: 30s)
- 비용 관리: 켈틱 크로스에는 clarifier 비허용 (이미 10장)

## 비용/성능 고려사항

- 조건 A, B, D: API 추가 호출 없음 (클라이언트/서버 계산)
- 조건 C: 기존 리딩 응답에 포함되므로 추가 호출 없음, 단 프롬프트 토큰 소폭 증가
- Clarifier 실제 실행 시: 추가 API 호출 1회 (원 카드 기준 ~500~800 토큰)
- 24시간 제한: clarifier는 기존 리딩 1회 제한에 포함시킬지 별도 카운트할지 결정 필요
